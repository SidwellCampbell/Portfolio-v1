PokeVault is a full-stack web application built using Java Spring Boot, showcasing my ability to integrate external APIs, build interactive user interfaces, and implement robust back-end functionality with data persistence and user authentication.

This project evolved from a command-line version of the app, where users could search for Pokémon and save their favorites to a text file using Java and HttpRequest from the PokeAPI. While that version taught me valuable lessons about API consumption and data handling, I wanted to elevate the project by adding a visual front-end component and leveraging a database for persistent storage. PokeVault allowed me to deepen my understanding of building full-stack applications, integrating external APIs, managing databases, and deploying applications in the cloud. PokeVault not only demonstrates technical skills but also my passion for learning and bringing ideas to life through software.

Key Features

  • Dynamic API Integration: The app fetches data from the PokeAPI, presenting detailed Pokémon information seamlessly on the front end.
  • Data Persistence: Uses a MySQL database to store user data and Pokémon favorites, ensuring data is saved across sessions.
  • Authentication & Authorization: Implements Spring Security, requiring users to log in to access specific features and ensuring secure handling of user data.
  • AWS Deployment: The application is deployed on AWS, utilizing Elastic Beanstalk for hosting and AWS RDS for database management.
